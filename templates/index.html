🔬 STATIC ANALYSIS ENGINE

Scans every line of your source code using regex pattern matching. Detects RSA, ECC, MD5, SHA1, Diffie-Hellman, Weak TLS, and more. Returns exact file path and line number for each finding.

scanner.py → scan_file() → scan_text_with_lines() → Returns: { line_number, vulnerability_type, risk }
🤖 RANDOM FOREST ML MODEL

Trained on a synthetic 60-sample dataset. Takes an 8-feature binary vector as input and outputs a risk class with probability scores. Uses 5-fold cross-validation for reliable accuracy.

Features: [RSA, ECC, MD5, SHA1, DH, TLS, AES, PQC] → Output: Low / Medium / High + confidence %
⚡ RISK ENGINE + SCORING

Rule-based classifier assigns CRITICAL / HIGH / MEDIUM / LOW per file. Score starts at 100 and deducts penalty points per vulnerability type found. Overall project score calculated across all files.

RSA/ECC/DH → -25 pts each (CRITICAL) MD5/SHA1/TLS → -15 pts each (HIGH) Score = max(0, 100 - penalties)

What QuantumReady Detects

8 cryptographic vulnerabilities detected — ranked by quantum threat level. Each comes with the exact NIST-approved replacement.

💀 CRITICAL — Broken by Shor's Algorithm (Quantum)
RSA CRITICAL

RSA encryption is fully broken by Shor's Algorithm on a sufficiently large quantum computer. All key sizes are vulnerable.

❌ VULNERABLE
from Crypto.PublicKey import RSA key = RSA.generate(2048)
✅ QUANTUM-SAFE
import oqs kem = oqs.KeyEncapsulation("Kyber512") pub = kem.generate_keypair()
🔐 NIST FIPS 203 (CRYSTALS-Kyber)
ECC CRITICAL

Elliptic Curve Cryptography including secp256k1 and NIST P-curves is broken by Shor's Algorithm. Used widely in TLS and Bitcoin.

❌ VULNERABLE
from Crypto.PublicKey import ECC key = ECC.generate(curve="P-256")
✅ QUANTUM-SAFE
import oqs sig = oqs.Signature("Dilithium3") pub = sig.generate_keypair()
🔐 NIST FIPS 204 (CRYSTALS-Dilithium)
Diffie-Hellman CRITICAL

DH key exchange relies on the discrete logarithm problem, which Shor's Algorithm solves efficiently on quantum hardware.

❌ VULNERABLE
DiffieHellman.generateKey() # or DHKeyPairGenerator usage
✅ QUANTUM-SAFE
kem = oqs.KeyEncapsulation("Kyber768") pub = kem.generate_keypair() ct, ss = kem.encap_secret(pub)
🔐 NIST FIPS 203 (Kyber KEM)
🔴 HIGH — Broken on Classical Computers Today
MD5 HIGH

MD5 has known collision attacks — two different inputs can produce the same hash. Never use for passwords or integrity checks.

hashlib.md5(data).hexdigest() hashlib.sha3_256(data).hexdigest()
🔐 NIST FIPS 202 (SHA-3)
SHA-1 HIGH

SHA-1 collisions were demonstrated in 2017 (SHAttered attack). Chrome, Firefox, and CAs have deprecated it. Still found in legacy code.

hashlib.sha1(data).hexdigest() hashlib.sha3_256(data).hexdigest()
🔐 NIST FIPS 202 (SHA-3)
Weak TLS HIGH

TLS 1.0 and TLS 1.1 are deprecated by RFC 8996. Vulnerable to POODLE, BEAST, and multiple downgrade attacks.

ssl.PROTOCOL_TLSv1 # deprecated ctx.minimum_version = TLSVersion.TLSv1_3
🔐 NIST SP 800-52 Rev. 2
⚠️ MEDIUM / LOW — Weakened by Grover's Algorithm
KeyPairGenerator MEDIUM

Java's KeyPairGenerator detected — risk depends on the algorithm used. If RSA or EC, it is CRITICAL. Must be reviewed and replaced.

AES LOW

Grover's Algorithm halves AES key strength. AES-128 → 64-bit quantum security (too weak). AES-256 → 128-bit security (acceptable). Always use AES-256-GCM.

The Quantum Threat

Shor's Algorithm (1994) proves that quantum computers can factor large numbers and solve discrete logarithms exponentially faster than classical computers — breaking RSA, ECC, and DH instantly.

🏛️

NIST Response (2024)

In August 2024, NIST published three post-quantum standards: FIPS 203 (Kyber — key exchange), FIPS 204 (Dilithium — signatures), and FIPS 205 (SPHINCS+ — hash signatures).
